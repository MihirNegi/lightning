# RUST_REPO_CONTEXT — `C:\repositories\rust`

Deep scan of the `rust` repository. This is a **teaching demo** that mirrors Prime Video's architecture: the **entire UI** is written in Rust, compiled to WebAssembly, and drawn to a single `<canvas>` via WebGL2. No DOM is used for UI. JavaScript is a thin glue layer.

---

## 1. High-level summary

- **Purpose:** learn / prove that a full streaming-app UI (Home / Meta / Player, tabs, hero, rails) can be built with 100% Rust drawing to one WebGL2 canvas, with no React, no DOM UI, and only vanilla-JS glue.
- **Target:** desktop Chrome (primary), Samsung Tizen TVs (secondary — the code handles the TV Back key `keyCode 10009` and the Tizen `exit()` API).
- **Status:** milestones M0 → M7 all done; plus HERO banner, TEXT rendering, and paginated tabs added.
- **Not a product:** DRM / adaptive streaming (Shaka, HLS, Widevine) is explicitly out of scope for this phase — see PRD §12 for the future plan.
- **How to run:** `wasm-pack build --target web` (produces `pkg/`) → `python3 -m http.server 8000` → open `http://localhost:8000`. **No Node build.** The local Node is v11 which is why Vite isn't used.

## 4. Architecture — division of labour

```
+--------------------------------------------------+
| Browser tab                                       |
|                                                   |
| +----------+  keydown       +---------------+     |
| | JS glue  | -------------> |  Rust / Wasm  |     |
| | (thin)   |                |    engine     |     |
| |          | <------------- |               |     |
| | - rAF    |  draw cmds     | - state M/C   |     |
| | - input  |   (WebGL2)     | - layout      |     |
| | - image  |                | - animation   |     |
| |   decode |                | - scene/draw  |     |
| | - fetch  |                | - player UI   |     |
| | - video  |                +---------------+     |
| |   ctrl   |                       |               |
| +----+-----+                       v               |
|      |  textures / play    +------------------+   |
|      v                     |  <canvas> WebGL2 |   |
|  +-----------+  layered    |  (UI on top of   |   |
|  | <video>   | <--------->  |   the video)    |   |
|  | (mp4)     |             +------------------+   |
|  +-----------+                                     |
+--------------------------------------------------+
```

**Rust owns** the app state machine (Home / Meta / Player), layout math, focus model, easing/animation, the entire scene graph, all WebGL2 draw calls, and the player overlay.

**JS glue owns only host-reachable things:** the `requestAnimationFrame` loop, keydown forwarding, image decode (`createImageBitmap` → `ImageBitmap` handed straight to Rust), `fetch()` of `content.json` + tab pages, text rasterisation via a hidden 2D canvas → RGBA → texture, and driving the native `<video>` element.

**Video** plays in a native `<video>` behind the canvas. On the Player screen the canvas clears with `alpha = 0` so the video shows through.

## 7. Screens (the state machine)

```
   ┌──────┐   Enter   ┌──────┐   Enter   ┌────────┐
   │ Home │──────────►│ Meta │──────────►│ Player │
   │      │◄──────────│      │◄──────────│        │
   └──────┘  Back     └──────┘  Back     └────────┘
```

Rust enum `Screen { Home, Meta, Player }` (line ~184). Transitions:

- **Home → Meta:** JS pre-renders `meta:title` + `meta:desc` textures for the focused item, then calls `enter_meta()`.
- **Meta → Player:** JS points `<video>.src = engine.focused_video()`, calls `video.play()` (with a muted-retry fallback if autoplay is blocked), then `enter_player()`.
- **Any → back one level:** `Escape` / `Backspace` on laptop, TV keyCode `10009`, calls `engine.back()`. On Player it also stops the video (`pause + removeAttribute('src') + load()`). On Home top-level, it invokes `tizen.application.getCurrentApplication().exit()` if available (no-op in normal browser).
- **Focus is never mutated by transitions** — coming back from Player→Meta→Home lands exactly where you were.

### 7.1 Home zones

Home is further split into three focus **zones**: `Zone { Tabs, Hero, Rails }` (line ~150).

- **Tabs zone** — top: Left/Right cycle tabs (pointer-swap of cached content; skeleton if data not yet loaded); Down enters Hero (or Rails if this tab has no hero).
- **Hero zone** — the top 70% featured banner (`HERO_FRAC = 0.7`). Left/Right cycle the carousel (with a slide transition, eased `HERO_SLIDE_TAU_MS = 180ms`). Auto-advances every `HERO_AUTO_MS = 5000ms` while focused. Up returns to Tabs; Down enters Rails. Enter = "More Info" — points focus at the hero's referenced `(rail, card)`, decodes that poster if needed, then opens Meta.
- **Rails zone** — the poster grid. Arrows navigate; Up from rail 0 returns to Hero (or Tabs if no hero); Enter opens Meta for the focused card.

### 7.2 Meta

Layout: large poster on left (`ch * 0.62` tall, 2:3 aspect), title + wrapped description on the right, "▶ Play" pill (always highlighted since it's the only affordance).

### 7.3 Player

- Canvas clears with **alpha = 0** so the native `<video>` shows through.
- Rust draws: title top-left (re-using `meta:title` texture), bottom scrim, progress bar (fill = `currentTime / duration`), play icon (`ui:icon_play` texture) when paused / two-bar pause primitive when playing.
- JS pushes `set_playback(current, duration, paused)` on every frame while on Player.
- Enter toggles `video.play()` / `video.pause()`. Back stops + returns to Meta.

## 8. `main.js` responsibilities (thin JS glue, 370 lines)

1. `import init, { Engine } from "./pkg/app.js"` and instantiate the wasm module.
2. **Resize**: on window resize call `engine.set_size(innerWidth, innerHeight)`.
3. **Bootstrap textures** — render static UI text with `makeTextTexture()`:
   - `ui:play` (▶ Play), `ui:icon_play` (▶), `ui:moreinfo` (More Info).
4. **Fetch manifest** `content.json` → set `engine.set_tab_count(n)`, render `tab:i` (white) and `tabd:i` (dark) textures per tab.
5. **Lazy tab fetch** — `ensureTabFetched(i)` starts a `fetch(tab.file).then(r.text())` and stashes text in `pendingTexts`. The **parse** is deferred until `!engine.scroll_busy()` — the parse never blocks a slide animation. Parsed tabs are memoized inside the engine (`tabs_parsed: Vec<Option<Content>>`), so revisit = instant pointer swap.
6. **Incremental rail-title rendering** — on tab switch, `queueRailTitles()` queues indices; `drainTitles()` renders **4 titles per frame** (rendering 20-25 at once dropped a frame).
7. **Hero assets** — `renderHeroAssets()` renders `hero:title:i` (96px bold) and `hero:desc:i` (32px wrapped paragraph) textures, and eagerly `decodeBitmap()`s each featured backdrop.
8. **Lazy poster loading with throttled uploads**:
   - `engine.posters_to_load()` returns URLs in the 2D window that aren't on the GPU.
   - `decodeBitmap(url)` (fetch → blob → `createImageBitmap`) — any number in flight.
   - `uploadQueue[]` collects decoded bitmaps.
   - `drainUploads()` uploads **at most `MAX_UPLOADS_PER_FRAME = 2`** per frame — bursty `tex_image_2d` calls used to stretch a frame on the TV.
   - Runs `pumpPosters()` every ~150 ms + `engine.evict_lru()` to bound texture count.
9. **Text rendering** — done in JS by drawing to a hidden 2D canvas then reading back pixels:
   - `makeTextTexture(text, opts)` — single-line, returns `{width, height, pixels}`.
   - `makeParagraphTexture(text, {maxWidth, fontPx, lineHeight, weight, color})` — word-wraps to multiple lines. Used for hero desc + Meta desc.
10. **Player control** — hides/shows `<video>`, loads `src`, `play()`/`pause()`, and pushes `set_playback` into Rust every frame on the Player screen.
11. **Keyboard routing** — one `keydown` handler. Handles Tizen `keyCode 10009` first (Back), then `ArrowUp/Down/Left/Right` (→ `engine.input(e.key)` — Rust ignores unless on Home), `Enter` (context-sensitive: Home→Meta from hero or card, Meta→Player, Player→toggle play), `Escape`/`Backspace` (`goBack()`).
12. **rAF loop `frame(now)`**:
    - `engine.tick(dt)` — advances animation and draws.
    - Times `tick()` with `performance.now()` around it to compute `work-ms`; averages over ~300 ms and uploads `ui:fps` (`"NN fps  X.X ms  work Y.Y ms"`) which Rust draws top-right.
    - **All texture work is gated on `!engine.scroll_busy()`** — so decodes/uploads/parses only happen when motion has settled. Pure animation frames stay smooth.

### 9.1 Tuning constants

| Const                    | Value  | What it does                                          |
| ------------------------ | ------ | ----------------------------------------------------- |
| `SMOOTH_TAU_MS`          | 90.0   | Exponential-smoothing time constant (scroll + card pop). Smaller = snappier. |
| `FOCUS_SCALE_MAX`        | 1.10   | Focused card grows 10%.                              |
| `HERO_FRAC`              | 0.7    | Hero banner occupies top 70% of viewport.            |
| `HERO_AUTO_MS`           | 5000.0 | Auto-advance carousel every 5 s.                     |
| `HERO_SLIDE_TAU_MS`      | 180.0  | Hero panel slide time (slower than scroll).          |
| `TAB_BAR_H`              | 64.0   | Fixed top tab-bar height in px.                      |
| `LOAD_RADIUS`            | 2      | Load posters for rails within ±2 of focus.           |
| `HSPAN`                  | 6      | Per rail, load ±6 cards around that rail's focus.    |
| `SETTLE_PX`              | 40.0   | Distance from target below which motion is "settled".|
| `MAX_POSTER_TEXTURES`    | 180    | LRU cap on resident poster/backdrop textures (~95 MB @ 540 KB each). |
| `TITLE_H`, `TITLE_GAP`   | 38, 30 | Rail title text height & gap below title.            |
| `MAX_UPLOADS_PER_FRAME`  | 2 (JS) | GPU uploads drained per rAF frame.                   |

### 9.2 Animation model — exponential smoothing

Each frame: `current += (target - current) * (1 - exp(-dt_ms / TAU))`. Naturally ease-out and roughly frame-rate independent. Applied to `anim_y` (vertical scroll), `anim_x[r]` (per-rail horizontal scroll), `focus_scale`, `hero_anim`, `tab_slide`.

`focus_scale` is reset to `1.0` on **a real focus move** (not at edges) so the pop plays only when focus actually changes.

### 9.3 Layout metrics (`metrics(ch)` — `src/lib.rs:775`)

```rust
card_h = ch * 0.34
card_w = card_h * 2.0 / 3.0        // posters are 2:3
card_pitch = card_w + 18.0
rail_pitch = TITLE_H + TITLE_GAP + card_h + 36.0
margin_x = 64.0, margin_y = 60.0
```

Vertical scroll target: `Zone::Tabs | Zone::Hero → 0.0`, `Zone::Rails → hero_h + focus_rail * rail_pitch`. Focus frame is **fixed** at a resting slot (rails slide under a stationary cursor). This is why edge presses don't blink.

### 9.4 Draw order (per frame, `draw()` at `src/lib.rs:785`)

1. `clear` — RGBA `(0.07, 0.07, 0.10, 1.0)` on Home/Meta; **`(0,0,0,0)` on Player** (transparent).
2. `use_program` + upload `u_resolution`, sampler.
3. Dispatch by `Screen`:
   - **Home**: rails (deferring the focused card), focused card **on top** (scaled by `focus_scale`), fixed focus frame (5px, cyan `[0.30, 0.80, 1.0, 1.0]`), hero (with slide transition), tab bar last (occludes rails scrolled up under it).
   - **Meta**: large poster, title (shrinks to fit column if long), wrapped desc, Play pill.
   - **Player**: bottom scrim, title top-left, progress track + fill, play icon (paused) OR two-bar pause primitive.
4. FPS readout — `ui:fps` texture, top-right (48 px tall).

### 9.5 Rendering primitives

- **`fill_rect(x,y,w,h,color)`** — sets `u_use_tex = 0`, uploads `u_color`, draws.
- **`blit_texture(x,y,w,h,tex)`** — sets `u_use_tex = 1`, binds `TEXTURE0`, draws.
- **`draw_rect`** — uploads 6 vertices (2 triangles, 4 floats each = pos.xy + uv.xy) into `vbo` with `DYNAMIC_DRAW` and calls `draw_arrays(TRIANGLES, 0, 6)`. **One draw call per quad — no true batching yet** (deferred; noted in CLAUDE.md M1).
- **`draw_frame`** — hollow rectangle drawn as 4 thin `fill_rect`s.
- **`draw_card`** — texture if resident, else grey placeholder `[0.6, 0.6, 0.62, 1.0]`.
- **Shaders** (embedded in `lib.rs:59-88`):
  - Vertex: `a_pos` (px) → clip space via `u_resolution` (with Y-flip), passes `a_uv`.
  - Fragment: two modes — `u_use_tex == 1` sample `u_tex`, else fill `u_color`.
- **Blending**: `SRC_ALPHA, ONE_MINUS_SRC_ALPHA` enabled globally.
- Texture params: `LINEAR/LINEAR`, `CLAMP_TO_EDGE` on both axes.

### 9.6 Texture uploads — two paths

- **`add_poster_bitmap(url, ImageBitmap)`** — uploads a decoded ImageBitmap **directly** via `webgl2.tex_image_2d_with_u32_and_u32_and_image_bitmap(...)`. **No pixel readback, no ~0.5 MB byte copy** — much cheaper on the main thread (matters on the TV).
- **`add_poster(url, w, h, &[u8])`** — falls back to raw RGBA bytes; used for text textures which need canvas 2D readback anyway.
- **`add_text_texture(key, w, h, pixels)`** — same as `add_poster` under the hood; `upload()` (line 598) frees any prior texture under that key before creating the new one — so re-uploading a key (fps readout, meta:title on Meta re-entry) never leaks.

### 9.7 Lazy loading / memory management

Three cooperating mechanisms:

1. **2D window** — `window_posters(radius)` returns the set of poster URLs for rails within ±`LOAD_RADIUS` of `focus_rail`, and per rail only cards within ±`HSPAN` of that rail's focused card.
2. **`posters_to_load()`** — `window_posters(LOAD_RADIUS) − textures.keys()` — JS pumps this list.
3. **`evict_lru()`** — when `total > MAX_POSTER_TEXTURES`, evict oldest (`t.used < …`) **except** the current window and hero backdrops (protected).
4. **Rail virtualisation in `draw_home()`** — culls rails and cards that are off-screen. 50 rails ≈ same per-frame work as 5.
5. **On every `tick()`** — stamps `frame_seq` onto textures currently in the window (their `used` is bumped, so they can't be evicted while they're on-screen).

## 10. Pagination / tab loading choreography

Because Node is v11 (no Vite) and content lives in static JSON, tabs are loaded on demand:

1. `main.js` fetches `content.json` at boot → sets `engine.set_tab_count(n)`.
2. `ensureTabFetched(0)` immediately starts fetching `tab-0.json` (only the fetch — the parse waits).
3. **Every rAF frame**, if `!engine.scroll_busy()` and there are unparsed tab texts, JS calls `engine.load_tab(i, text)` for each. This happens off the slide, so `serde_json::from_str` never blocks animation.
4. If the tab is the active one, `load_tab` swaps `content` and calls `reset_layout(n)` (resets focus, `card_focus[]`, `anim_x[]`, hero). Otherwise it's stored in `tabs_parsed[i]`.
5. On tab switch (`ArrowLeft/Right` in Tabs zone), `activate(i)` does a **pointer swap** — outgoing tab's `content` moves into `tabs_parsed[old]`, incoming `tabs_parsed[new].take()` becomes `content`. No parse. **Cached tabs re-appear instantly.**
6. If the target tab isn't parsed yet, `content = None` → `draw_home()` renders a **grey skeleton** (line 971) instead. JS's `ensureTabFetched(i)` kicks off the fetch; once parsed on the next settled frame, titles + heroes render.
7. `append_rails(json)` also exists (`src/lib.rs:463`) for future paginated rail fetching — extends `content.rails`, `card_focus[]`, `anim_x[]`. Not currently wired in this build (each tab is a single file), but the plumbing (`loaded_rail_count`, `focus_rail`, `scroll_busy`) is there.

## 13. Milestone history (from `CLAUDE.md` — all done)

- **M0** — Toolchain + one WebGL2 quad on canvas.
- **M1** — Poster textures; image → RGBA → GPU handoff proven. One draw call per poster (true sprite batching still deferred).
- **M2** — `fetch + serde_json`; 10 rails × 10 items; rail 0 rendered data-driven.
- **M3** — All 10 rails rendered; per-rail focus memory (`card_focus`); arrow-key nav; instant scroll keeps focus in view.
- **M4** — rAF `tick(dt)`; scroll eased via exponential smoothing; focused card pop, drawn on top.
- **M4a** — Grey placeholders + dynamic poster loading; fixed focus frame (rails slide under a stationary cursor); no blink at edges.
- **TEXT** — Text via JS `fillText` → RGBA → texture. Rail titles drawn.
- **M5** — Meta / details page. Screen enum {Home, Meta}. Enter → Meta; Escape → Home. Focus preserved.
- **M6** — Player. Native `<video>` behind a **transparent** canvas. Rust draws overlay (title, progress, play/pause). Sample mp4 = Big Buck Bunny.
- **HERO** — Top featured banner (`HERO_FRAC = 0.7`) with backdrop + scrim + title + synopsis + More Info + dots. Auto-slide 5s. `Zone` enum. Slide transition with `hero_prev + hero_anim`. Backdrop URLs whitelisted in `free_offscreen`.
- **M7 polish** — Frame-time readout top-right. Lazy posters: `posters_to_load()` / eviction windowed at `±LOAD_RADIUS`; free beyond `FREE_RADIUS`. `upload()` frees prior texture under a key (no leak on re-upload). **Note:** fps reads wrong in a backgrounded/preview tab (rAF throttled).
- **Scale extension** — Paginated tabs (this repo): 3 tabs × 15/20/25 rails × ~10 cards; each tab lazy-fetched, lazy-parsed (deferred to settled frames), then pointer-swapped. Texture 2D window `LOAD_RADIUS × HSPAN`. Rail virtualisation in draw = ~constant per-frame cost with any rail count.

## 14. Performance strategies (why this stays smooth)

| Problem                                    | Solution                                                                 |
| ------------------------------------------ | ------------------------------------------------------------------------ |
| ~100 posters × 540 KB would pressure GPU   | LRU + 2D window (`LOAD_RADIUS = 2`, `HSPAN = 6`), cap 180 textures       |
| `tex_image_2d` bursts stretch frames on TV | `MAX_UPLOADS_PER_FRAME = 2` in JS drainUploads                            |
| Tab-switch parsing blocks the slide         | Fetch immediately, **parse only when `!scroll_busy()`**                   |
| 20-25 rail-title text renders dropped a frame | Incremental `drainTitles()` — 4 per frame                              |
| Rendering all 50 rails would waste GPU     | Rail + card culling in `draw_home()`                                     |
| Re-uploading `ui:fps` every 300 ms leaked   | `upload()` deletes any prior texture under the same key                  |
| JS pixel readback for posters is heavy     | `add_poster_bitmap()` uploads an `ImageBitmap` straight to GL             |
| Focus pop "blinks" at edges                 | Reset `focus_scale = 1.0` only when `(rail, card)` actually changed      |
| Backdrops kept getting evicted             | `evict_lru()` protects the featured backdrop URLs explicitly              |
| Player canvas hides the video               | Player screen clears with `alpha = 0`                                     |

## 15. Known limitations & callouts

- **No true sprite batching** — one draw call per quad. CLAUDE.md flags this as deferred.
- **`tick_work` FPS reads wrong** in a backgrounded/preview tab because rAF is throttled by the browser. Foreground readouts are accurate.
- **Autoplay** may be blocked if Enter is a synthetic event; JS retries `muted`.
- **Old GTV bucket mp4** returns 403; the repo uses Blender's Big Buck Bunny mirror.
- Node stays at v11 — do not introduce a Vite/npm build step. Serve with `python3 -m http.server`.
- `pkg/` is generated by wasm-pack; never hand-edit.
- Assets (`poster1..poster6.png`) are bundled but not currently referenced by the tab JSONs (they use `picsum.photos` URLs). Likely leftover from an earlier milestone.
